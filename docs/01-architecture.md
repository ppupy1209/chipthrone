# 아키텍처 설계

## 1. 시스템 개요

chipthrone은 삼성전자(005930)와 SK하이닉스(000660)의 주가·시가총액을 실시간 비교하는 단일 페이지 대시보드다.
핵심 제약은 **AWS 프리티어 1대(EC2 t3.micro)** 에서 동작해야 한다는 것.

```
┌──────────────┐   SSE    ┌─────────────────────────────┐
│  React SPA   │◀─────────│  Spring Boot (EC2 t3.micro) │
│  (Vercel)    │  HTTP    │  ├ Scheduler (단일 폴링)     │
└──────────────┘─────────▶│  ├ Caffeine Cache           │
                          │  ├ SSE Emitter (fan-out)    │
                          │  └ Resilience4j (서킷브레이커)│
                          └──────┬──────────────┬────────┘
                                 │              │
                       ┌─────────▼───┐   ┌──────▼────────┐
                       │ RDS MySQL   │   │ 외부 데이터 소스 │
                       │ db.t3.micro │   │ KIS / Hyperliquid│
                       └─────────────┘   └───────────────┘
```

## 3. 핵심 과제: 외부 API 호출 최소화 (fan-out)

문제: 유저 요청마다 외부 API를 호출하면 (1) 무료 쿼터 소진 (2) 응답 지연 (3) 장애 전파.

해결:
1. `@Scheduled` 워커가 **주기(예: 1~3초)마다 한 번만** 외부 시세를 폴링한다.
2. 최신 시세를 **Caffeine 인메모리 캐시**에 저장한다. (프리티어라 Redis 미사용)
3. 클라이언트는 **SSE**로 구독하고, 폴링 결과를 N명에게 fan-out 한다.
4. 외부 API 실패 시 **Resilience4j 서킷브레이커** + 마지막 캐시값 폴백.

→ 동시 접속자가 늘어도 외부 호출 횟수는 일정하게 유지된다.

## 4. 데이터 모델 (외래키 · JPA 연관관계 미사용)

운영 환경 관행에 따라 DDL에 FK 제약을 두지 않고, 엔티티 간 연관관계 매핑(`@ManyToOne` 등)도 사용하지 않는다.
참조는 ID 값으로만 하고, 조인이 필요하면 애플리케이션 레벨에서 처리한다.

- `company` — 종목 마스터 (종목코드, 이름, 상장주식수)
- `price_snapshot` — 시점별 시세 (company_id 값 참조, FK 없음)
- `rank_flip_event` — 시총 1·2위 역전 발생 기록 (재미 기능 / 역사 타임라인용)

## 5. 시가총액 & 역전 계산

```
시가총액 = 현재가 × 상장주식수
격차(%)  = (1위 시총 − 2위 시총) / 2위 시총 × 100
역전 임계 등락률 = (1위 시총 / 2위 상장주식수 / 2위 현재가 − 1) × 100
```

상장주식수는 자주 바뀌지 않으므로 `company` 테이블에 저장하고 주기적으로 갱신한다.

## 6. 배포 (CI/CD)

1. `main` 머지 시 GitHub Actions 트리거.
2. 백엔드: Gradle 빌드 → Docker 이미지 → EC2에 배포(SSH/compose).
3. 프론트엔드: Vercel 자동 배포.
4. Nginx가 리버스 프록시 + Let's Encrypt SSL 담당.
